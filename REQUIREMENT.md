# 반도체 시료 생산주문관리 시스템 - 요구사항 명세

> 출처: `[CRA_AI] Day3_개인과제_반도체시료관리.pdf`
> 구현 방식(자료구조/알고리즘 등 상세 설계)은 [design.md](./docs/DESIGN/design.md)를 참조한다.

## 1. 배경

가상의 반도체 회사 "S-Semi"는 다양한 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체,
대학 연구실 등에 납품한다. 시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐
고객에게 출고된다.

기존에는 엑셀과 메모장으로 주문을 관리하여 실수가 잦고, 재고와 공정 현황을 한눈에 파악하기 어려운
문제가 있었다. 이를 해결하기 위해 "반도체 시료 생산주문관리 시스템"을 개발한다.

## 2. 역할별 흐름도

```
고객 --(시료 요청)--> 주문 담당자 --(주문서 전달)--> 생산 담당자
                                    <--(승인/거절)--
```

- 고객: 필요한 시료를 이메일 등으로 요청
- 주문 담당자: 요청에 맞게 주문서 작성 (주문서 관리)
- 생산 담당자: 개발 시료 등록, 주문 수신 후 승인 또는 거절 (시료 생산·승인)

## 3. 시스템 개요

- 생산 라인: 공장에서 시료 하나를 생산하는 설비 흐름
  - 하나의 생산 라인은 시료를 하나씩 생산
  - 주문이 들어온 시료에 대해서만 생산
- 시스템은 **콘솔 기반**으로 동작하며, 담당자가 직접 명령을 입력하여 시료를 등록하고 주문을 처리

### 전체 흐름 (상태 다이어그램)

```
주문 등록 (RESERVED)
   │
   ▼
승인 여부? ──거절──> REJECTED
   │ 승인
   ▼
재고 확인?
   │재고 충분              │재고 부족
   ▼                       ▼
출고 준비 (CONFIRMED) <── 생산 요청 (PRODUCING)
   │                       (생산 완료 시 CONFIRMED로 전환)
   ▼
출고 처리 (RELEASED)
   │
   ▼
고객 도달
```

> 주의: `PRODUCING`은 생산 완료 후 **반드시 `CONFIRMED`를 거쳐야** `RELEASED`(출고)로 갈 수 있다.
> `PRODUCING → RELEASED` 로 바로 전환되는 흐름은 없다.

## 4. 주문 상태

모든 주문은 아래 상태 중 하나를 가진다. `REJECTED`는 정상 흐름 외의 상태이며 모니터링에서 제외한다.

| 상태 | 의미 |
|---|---|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 |
| PRODUCING | 주문 승인 완료 및 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 및 출고 대기 중 |
| RELEASED | 출고 완료 |

## 5. 기능 명세

### 5.1 메인 메뉴

- 기능별 선택 화면 표시
- 전체 시료에 대한 요약 정보 확인

| 메뉴 | 의미 |
|---|---|
| 시료 관리 | 새로운 시료 등록, 목록 조회, 시료명 검색 기능 |
| 주문 (접수/승인/거절) | 고객 주문 접수 및 생산 라인 담당자의 승인·거절 처리 |
| 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 출고 처리 | CONFIRMED 상태 주문에 대해 출고 실행 |
| 생산 라인 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |

### 5.2 시료 관리

시료(Sample)는 시스템의 가장 기본이 되는 단위. 각 시료는 고유한 시료명과 속성을 가지며,
시스템에 등록된 시료만 주문 가능하다.

| 메뉴 | 의미 |
|---|---|
| 시료 등록 | 새로운 시료를 시스템에 추가 (속성: 시료 ID, 시료명, 평균 생산시간, 수율) |
| 시료 조회 | 등록된 모든 시료 목록과 현재 재고 수량을 확인 |
| 시료 검색 | 시료명 등 속성으로 특정 시료를 검색 |
| 뒤로가기 | 메인 메뉴로 복귀 (PDF 예시 UI 기준 `[0] 뒤로`) |

- 수율 = 정상적인 시료 수 / 총 생산 시료 수 (예: 100개 생산 중 정상 90개 = 0.9)
- 평균 생산시간은 개당 생산시간이다

### 5.3 시료 주문

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

- **시료 예약**: 고객이 원하는 시료와 수량을 주문. 이 시점의 주문 상태는 `RESERVED`
- 예약 시 입력 값: 시료 ID, 고객명, 주문 수량
- 이 단계에서 생성되는 주문은 항상 `RESERVED` 상태이며, 승인(`CONFIRMED`/`PRODUCING`) 또는
  거절(`REJECTED`) 여부는 **주문 담당자가 아닌 생산 담당자**가 다음 단계(5.4 주문 승인/거절)에서 결정한다.
- **주문 완료 출력**: 시료 예약(주문 생성)이 완료되면 아래 정보를 포함한 주문 내역을 화면에 출력한다.
  - 주문번호, 시료(ID/시료명), 고객명, 주문 수량, 현재 상태(`RESERVED`)

> 구현 참고: `RESERVED` 주문의 큐잉/처리 순서(FIFO) 등 상세 구현 방식은
> [design.md - 주문 접수 큐](./docs/DESIGN/design.md#1-주문-접수-큐-reservation-queue) 참조.

### 5.4 주문 승인/거절

- **접수된 주문 목록**: `RESERVED` 상태의 주문 목록 표시
- **주문 승인**: 접수된 특정 주문에 대해 승인. 재고 상황에 따라 자동 분기 처리
  - **재고 파악 출력**: 승인 처리 시 분기 판단에 앞서, 해당 시료의 현재 재고와 주문 수량을
    비교한 재고 파악 결과를 화면에 출력한다. (시료명, 현재 재고, 주문 수량, 충분/부족 여부,
    부족 시 부족분 수량)
  - 재고 충분 → 즉시 `CONFIRMED` 상태로 전환
  - 재고 부족 → 생산 라인에 자동 등록, 주문 상태를 `PRODUCING`으로 전환
- **주문 거절**: 접수된 특정 주문에 대해 거절, 즉시 `REJECTED` 상태로 전환
- **처리 결과 출력**: 승인/거절 처리가 끝나면 아래 정보를 포함한 결과를 화면에 출력한다.
  - 주문번호, 상태 변경 내역(`RESERVED` → `CONFIRMED`/`PRODUCING`/`REJECTED`)
  - 승인이고 재고 부족으로 생산 라인에 등록된 경우, 재고 현황(현재 재고/주문 수량/부족분)과
    실 생산량·예상 생산 시간도 함께 출력한다.

### 5.5 모니터링

- **주문량 확인**: 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASED) 주문 목록 확인
  (`REJECTED`는 유효한 주문이 아니므로 제외)
- **재고량 확인**: 시료별 현재 재고 수량 확인. 주문 대비 재고 수량에 따라 상태 표기
  - 여유: 주문 대비 재고 충분
  - 부족: 주문 대비 재고 수량 부족
  - 고갈: 재고 수량이 0

### 5.6 생산 라인

- 주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여 시료를 생산
  - 실 생산량 = `ceil(부족분 / 수율)`
  - 총 생산 시간 = `평균 생산시간 * 실 생산량`
  - 생산 완료 시 주문 상태 `PRODUCING` → `CONFIRMED` 로 전환
- **생산 현황 표기**: 현재 생산 중인 시료에 대한 정보 표기 (표기 정보 수준은 자율 결정,
  예: 주문 정보, 현재까지의 생산량 등)
- **대기 주문 확인**: 생산 큐(대기열)에서 대기 중인 목록 출력
  - 스케줄링 전략: **FIFO** (선입선출)

### 5.7 출고 처리

- 재고가 충분해진 `CONFIRMED` 주문에 대해 출고 처리
- 특정 주문에 대해 출고 실행 → 주문 상태를 `RELEASED`(원문 표기: RELEASE)로 전환

> 참고: 챕터 2 표에서는 `RELEASE`, 상태 다이어그램에서는 `RELEASED`로 표기되어 있음.
> 구현 시 하나의 상태명으로 통일 필요 (`RELEASED` 권장).
